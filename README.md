# VaultX – Decentralized Password Vault (Lite)

VaultX is a secure, minimal, and hackathon-ready decentralized password manager where users store, retrieve, and decrypt passwords using blockchain and client-side AES encryption.

## 🚀 Setup Instructions

### 1. Smart Contract Deployment
1. Open [Remix IDE](https://remix.ethereum.org/).
2. Create a new file named `VaultX.sol` and paste the code from the `VaultX.sol` file in this project.
3. Compile the contract using Solidity version `0.8.x`.
4. Go to the "Deploy & Run Transactions" tab.
5. Select "Injected Provider - MetaMask" as the Environment.
6. Deploy the contract to a testnet (e.g., Sepolia, Polygon Amoy).
7. Copy the **Contract Address** after deployment.

### 2. Connect Frontend
1. Open `src/constants.ts` in this project.
2. Replace `CONTRACT_ADDRESS` with your deployed contract address.
3. Ensure the `CONTRACT_ABI` matches the one generated by Remix (it should already be correct if you used the provided Solidity code).

### 3. Run Locally
1. Install dependencies:
   ```bash
   npm install
   ```
2. Start the development server:
   ```bash
   npm run dev
   ```
3. Open the app in your browser (usually `http://localhost:3000`).

## 🛡️ Security Model
- **Client-Side Encryption**: Passwords are encrypted using AES-256 with a user-defined Master Password before they ever leave your browser.
- **Decentralized Storage**: Encrypted data is stored on the blockchain, ensuring no central authority can access or censor your data.
- **Zero-Knowledge**: The Master Password is never stored on-chain or in any database. Only you hold the key to your vault.

## 🛠️ Tech Stack
- **Frontend**: React, Tailwind CSS, Motion (for animations)
- **Blockchain**: ethers.js, Solidity
- **Encryption**: crypto-js (AES)
- **Icons**: Lucide React
